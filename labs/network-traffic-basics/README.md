# Network Traffic Basics

This lab focused on understanding how network traffic works and how analysts can inspect it. In a Security Operations Center environment, analyzing network traffic is an important skill for detecting suspicious or malicious activity.

## Topics Covered

Network packets  
Protocol communication  
Traffic inspection  
Basic packet analysis concepts

## Key Concepts

Network Traffic  
Network traffic refers to the flow of data between devices on a network. This communication happens through packets that travel between source and destination systems.

Packet Structure  
Each packet contains a header and a payload. The header includes information such as source IP address, destination IP address and protocol type.

Protocols  
Protocols define how communication happens between systems. Examples include HTTP, HTTPS, DNS and TCP.

Traffic Analysis  
Security analysts inspect packets and traffic patterns to identify unusual behaviour such as suspicious connections, scanning activity or data exfiltration.

## Example

Tools like Wireshark allow analysts to capture and inspect packets moving through a network. By analyzing these packets an analyst can understand what systems are communicating and what type of data is being exchanged.
